# README for `iot/` Folder

## IoT - BSEC2 Configuration and AI-Studio Models

This folder contains the BSEC2 configuration files and trained AI models generated by Bosch BME AI-Studio. These files are used when running the professional Bosch workflow where machine learning models execute directly on the ESP32 sensor (edge AI), rather than in the cloud API.

## 🚀 **Overview**

This folder implements the **professional Bosch workflow** for the Wine Detector system. Instead of running machine learning models in the cloud API (as in the DIY approach), these files allow the models to run **directly on the ESP32** using the BSEC2 library.

### Key Difference from Training Folder

| Aspect | `iot/` Folder | `training/` Folder |
|--------|---------------|---------------------|
| **Purpose** | Edge AI (models on ESP32) | Cloud AI (models in API) |
| **Models** | Generated by BME AI-Studio | Trained with scikit-learn (KNN) |
| **Inference Location** | On the ESP32 sensor | In the Flask API |
| **File Format** | `.c` / `.h` (C source) | `.pkl` (Python pickle) |
| **Workflow** | Professional (Bosch) | DIY (manual) |

## 🔧 **How It Works**

### 1. Model Training (in BME AI-Studio)

The models in this folder are generated by **Bosch BME AI-Studio**, a desktop application for training neural networks on BME688 sensor data:

1. Collect labeled data using the BME688 development kit (SD card)
2. Import `.bmerawdata` files into AI-Studio
3. Define classes (air, toro, garnacha, monastrel, macabeo)
4. Train a neural network classifier
5. Export the trained model as `.c` and `.h` files

### 2. Integration with Firmware

The exported model files are placed in the appropriate subfolders (`presence_model/`, `type_model/`, etc.). The firmware (`wine_detector_bme688.ino`) includes these files and uses BSEC2 to run inference directly on the ESP32.

### 3. Hierarchical Execution on ESP32

```
┌─────────────────────────────────────────────────────────────────┐
│                         ESP32 Firmware                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Load presence_model (air vs wine)                           │
│     ↓                                                           │
│     If "wine" → Load type_model (red vs white)                  │
│     ↓                                                           │
│     If "red" → Load red_region_model (toro/garnacha/monastrel)  │
│     If "white" → Use white_region_model (macabeo)               │
│                                                                 │
│  2. Publish prediction via MQTT                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 📝 **File Descriptions**

### Model Subfolders (`presence_model/`, `type_model/`, `red_region_model/`, `white_region_model/`)

| File | Description |
|------|-------------|
| `bsec_config_*.h` | Binary configuration containing the trained neural network weights and parameters. Generated by BME AI-Studio. |
| `bsec_selectivity.c` | C source file implementing the selectivity model. Contains the model structure and inference functions. |
| `bsec_selectivity.h` | Header file declaring the model functions and data structures. |

### `bsec_config.h`

Main BSEC2 configuration file that defines which virtual sensors to enable:

```c
#ifndef BSEC_CONFIG_H
#define BSEC_CONFIG_H

#include <bsec2.h>

static const bsec_virtual_sensor_t SENSOR_LIST[] = {
    BSEC_OUTPUT_SENSOR_HEAT_COMPENSATED_TEMPERATURE,
    BSEC_OUTPUT_SENSOR_HEAT_COMPENSATED_HUMIDITY,
    BSEC_OUTPUT_RAW_PRESSURE,
    BSEC_OUTPUT_RAW_GAS,
    BSEC_OUTPUT_IAQ,
    BSEC_OUTPUT_GAS_ESTIMATE_1,   // For model predictions
    BSEC_OUTPUT_GAS_ESTIMATE_2,
    BSEC_OUTPUT_GAS_ESTIMATE_3,
    BSEC_OUTPUT_GAS_ESTIMATE_4
};

#endif
```

### `wine_detector_bme688.ino`

Arduino sketch that:
- Initializes all 8 sensors (using `commMux` for SPI/I2C multiplexing)
- Loads the hierarchical models from the subfolders
- Runs inference on each sensor
- Publishes predictions via MQTT

## 🏃 **How to Use**

### Prerequisites

- BME AI-Studio installed on Windows PC
- BME688 Development Kit (8 sensors)
- Labeled training data collected via SD card

### Steps to Generate Models

1. **Collect data** using the BME688 development kit (record to SD card)
2. **Import** the `.bmerawdata` files into BME AI-Studio
3. **Define classes** (air, toro, garnacha, monastrel, macabeo)
4. **Train** the neural network models (one per hierarchical level)
5. **Export** each model → copy `.c` and `.h` files to the corresponding subfolder

### Steps to Deploy

1. **Copy** the exported model files into the appropriate subfolders
2. **Open** `wine_detector_bme688.ino` in Arduino IDE or PlatformIO
3. **Configure** WiFi and MQTT settings in the sketch
4. **Upload** to ESP32
5. **Run** the MQTT subscriber and dashboard as usual
